# Database Layer

In the age of AI programming I want to be extra deliberate about what I do.
Why am I creating a database layer and what does it mean.  

This layer bases it's structure on the database's structure so if *(and when)* the database changes, 
I will need to change it here.

This is the layer I am the least familiar with. It is not part of or analogous to the MVC structure.  

This layer seems standard for Spring projects so I want to follow convention.
